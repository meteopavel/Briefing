# Chronicle Reporting Automation

📄🧠 Внутренний Python-инструмент для автоматизации отчётности по Redmine и постепенного ведения летописи выполненных задач.

## Что умеет сейчас

- загружает трудозатраты из Redmine за нужный период;
- формирует CSV с трудозатратами;
- генерирует акт в `.docx`;
- генерирует отчёт в `.docx`;
- использует локальные шаблоны документов;
- отделяет код проекта от локальных чувствительных данных.

## Установка зависимостей

Рекомендуется использовать виртуальное окружение Python 3.9.

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

## Запуск

Обычный запуск:
```
python main.py
```
Запуск с выводом расчётной таблицы в консоль:
```
python main.py --debug
```

## Логика работы

Сценарий:
1. определяется целевой месяц;
2. из Redmine загружаются трудозатраты;
3. timelog сохраняется в локальные данные;
4. генерируются акт и отчёт;
5. итоговые документы сохраняются в `.local_runtime/output/`.

## Безопасность хранения

В репозитории хранится:
- код проекта;
- безопасная инфраструктура проекта;
- зашифрованный архив локальных данных в `secure/sensitive_bundle.7z` при необходимости.

В репозитории не хранятся:
- `.env`;
- содержимое `.local_secure/` в открытом виде.

## Локальный deploy / backup workflow

Скрипт `deploy-local.sh` предназначен для локального сценария:
- обновляет зашифрованный архив локальных данных (если передать --archive);
- подготавливает изменения к коммиту;
- выполняет commit и push в GitHub.

## Планы развития

В дальнейшем проект будет расширяться в сторону:
- ведения структурированной летописи задач;
- накопления обезличенной базы коммерческого опыта;
- подготовки материалов для отчётности и портфолио.
